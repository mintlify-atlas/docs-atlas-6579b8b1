---
title: "SPIRE Agent CLI"
description: "Full command reference for the spire-agent binary."
---

The `spire-agent` binary provides commands for running the agent, interacting with the Workload API, and checking agent health. All `api` subcommands communicate with the agent over a Unix domain socket (default: `/tmp/spire-agent/public/api.sock`).

---

## `spire-agent run`

Starts the SPIRE Agent process.

| Flag | Description | Default |
|------|-------------|---------|
| `-allowUnauthenticatedVerifiers` | Allow the agent to release trust bundles to unauthenticated verifiers | |
| `-config` | Path to a SPIRE config file | `conf/agent/agent.conf` |
| `-dataDir` | Directory the agent can use for its runtime data | |
| `-expandEnv` | Expand `$VARIABLES` in the config file | |
| `-joinToken` | An optional join token generated by the SPIRE server | |
| `-joinTokenFile` | Path to a file containing a join token | |
| `-logFile` | File to write logs to | |
| `-logFormat` | Log format: `text` or `json` | |
| `-logLevel` | Log level: `DEBUG`, `INFO`, `WARN`, or `ERROR` | |
| `-serverAddress` | IP address or DNS name of the SPIRE server | |
| `-serverPort` | Port number of the SPIRE server | |
| `-socketPath` | Path to bind the Workload API socket | |
| `-trustBundle` | Path to the SPIRE server CA bundle | |
| `-trustBundleUrl` | URL to download the SPIRE server CA bundle | |
| `-trustDomain` | Trust domain this agent belongs to (max 255 chars) | |

```bash
spire-agent run -config /etc/spire/agent.conf
```

## `api` subcommands

The `api` subcommands are thin clients that call the SPIFFE Workload API exposed by the running agent. They are useful for testing and debugging SVID issuance.

### `spire-agent api fetch x509`

Fetches an X.509-SVID from the Workload API. This command is also aliased as `spire-agent api fetch`.

| Flag | Description | Default |
|------|-------------|---------|
| `-silent` | Suppress stdout output | |
| `-socketPath` | Path to the SPIRE Agent API socket | `/tmp/spire-agent/public/api.sock` |
| `-timeout` | Time to wait for a response | `1s` |
| `-write` | Write SVID data to the specified directory path (PEM files) | |

```bash
# Print SVID details to stdout
spire-agent api fetch x509

# Write SVID, key, and bundle PEM files to /tmp/svids/
spire-agent api fetch x509 -write /tmp/svids/
```

Example output:

```
Received 1 svid after 5.1234ms

SPIFFE ID:              spiffe://example.org/myservice
SVID Valid After:       2026-04-02 10:00:00 +0000 UTC
SVID Valid Until:       2026-04-02 11:00:00 +0000 UTC
CA #1 Valid After:      2026-04-01 00:00:00 +0000 UTC
CA #1 Valid Until:      2026-04-02 00:00:00 +0000 UTC
```

When `-write` is used, files are written as:

| File | Contents |
|------|----------|
| `svid.0.pem` | Leaf certificate (PEM) |
| `svid.0.key` | Private key (PKCS8 PEM) |
| `bundle.0.pem` | Trust bundle (PEM) |
| `federated_bundle.0.N.pem` | Federated trust bundle for federated domain N |

<Note>
  The process calling `spire-agent api fetch x509` must be attested by the agent and have a matching registration entry. The agent performs workload attestation using the calling process's OS identity.
</Note>

### `spire-agent api fetch jwt`

Fetches a JWT-SVID from the Workload API.

| Flag | Description | Default |
|------|-------------|---------|
| `-audience` | Comma-separated list of audience values (required) | |
| `-socketPath` | Path to the SPIRE Agent API socket | `/tmp/spire-agent/public/api.sock` |
| `-spiffeID` | The SPIFFE ID of the JWT being requested (optional; returns the default SVID if omitted) | |
| `-timeout` | Time to wait for a response | `1s` |

```bash
spire-agent api fetch jwt -audience myservice
spire-agent api fetch jwt -audience myservice -spiffeID spiffe://example.org/myservice
```

Example output:

```
token(spiffe://example.org/myservice):
	eyJhbGciOiJFUzI1NiIsImtpZCI6...

bundle(spiffe://example.org):
	{"keys":[{"kty":"EC","crv":"P-256",...}]}
```

### `spire-agent api validate jwt`

Validates a JWT-SVID by calling the Workload API's `ValidateJWTSVID` RPC.

| Flag | Description | Default |
|------|-------------|---------|
| `-audience` | Expected audience value (required) | |
| `-socketPath` | Path to the SPIRE Agent API socket | `/tmp/spire-agent/public/api.sock` |
| `-svid` | The JWT-SVID token string to validate (required) | |
| `-timeout` | Time to wait for a response | `1s` |

```bash
spire-agent api validate jwt \
  -audience myservice \
  -svid eyJhbGciOiJFUzI1NiIsImtpZCI6...
```

Example output (valid token):

```
SVID is valid.
SPIFFE ID : spiffe://example.org/myservice
Claims    : {"aud":["myservice"],"exp":1743600000,"iat":1743596400,"sub":"spiffe://example.org/myservice"}
```

If the SVID is invalid, the command exits with a non-zero status and prints:

```
SVID is not valid: ...
```

### `spire-agent api watch`

Attaches to the Workload API and watches for X.509-SVID updates, printing details whenever the SVID set changes (e.g., on rotation).

| Flag | Description | Default |
|------|-------------|---------|
| `-socketPath` | Path to the SPIRE Agent API socket | `/tmp/spire-agent/public/api.sock` |

```bash
spire-agent api watch
```

The command runs indefinitely until interrupted. Each time the SVID set is updated, it prints the new SVID details in the same format as `api fetch x509`.

## `healthcheck`

Checks the health of the SPIRE Agent process.

| Flag | Description | Default |
|------|-------------|---------|
| `-shallow` | Perform a less stringent health check | |
| `-socketPath` | Path to the SPIRE Agent API socket | `/tmp/spire-agent/public/api.sock` |
| `-verbose` | Print verbose information | |

```bash
spire-agent healthcheck
spire-agent healthcheck -verbose
```

A healthy agent exits with status `0` and prints:

```
Agent is healthy.
```

---

## `validate`

Validates a SPIRE Agent configuration file without starting the agent.

| Flag | Description | Default |
|------|-------------|---------|
| `-config` | Path to a SPIRE agent configuration file | `agent.conf` |
| `-expandEnv` | Expand `$VARIABLES` in the config file | `false` |

```bash
spire-agent validate -config /etc/spire/agent.conf
```
