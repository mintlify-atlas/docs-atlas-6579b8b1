The SPIRE Agent runs on each node and issues SVIDs to workloads. It attests to the SPIRE Server to obtain its own SVID, then listens on a Unix socket for workload API requests.

Configuration files may be written in HCL or JSON. Most file options have equivalent CLI flags that take precedence when set. Passing `-expandEnv` expands `$VARIABLE` and `${VARIABLE}` style environment variables before parsing — useful for templating config files or inserting secrets such as join tokens.

## Sample configuration file

```hcl
agent {
    trust_domain       = "example.org"
    trust_bundle_path  = "/opt/spire/conf/initial_bundle.crt"

    data_dir       = "/opt/spire/.data"
    log_level      = "DEBUG"
    server_address = "spire-server"
    server_port    = "8081"
    socket_path    = "/tmp/spire-agent/public/api.sock"
}

plugins {
    NodeAttestor "join_token" {
        plugin_data {}
    }
    KeyManager "disk" {
        plugin_data {
            directory = "/opt/spire/.data"
        }
    }
    WorkloadAttestor "unix" {
        plugin_data {}
    }
}
```

  <Tab title="HCL">

  </Tab>
  <Tab title="JSON">

## Agent configuration options

All options below live inside the top-level `agent { ... }` block.

| Option | Description | Default |
|---|---|---|
| `trust_domain` | The trust domain this agent belongs to. Should be no more than 255 characters. | — |
| `server_address` | DNS name or IP address of the SPIRE Server. | — |
| `server_port` | Port number of the SPIRE Server. | — |
| `socket_path` | Path to bind the SPIRE Agent API Unix socket (Unix only). | `/tmp/spire-agent/public/api.sock` |
| `data_dir` | Directory the agent uses for runtime data. | `$PWD` |
| `log_level` | Logging level: `DEBUG`, `INFO`, `WARN`, or `ERROR`. | `INFO` |
| `log_format` | Log output format: `text` or `json`. | `text` |
| `log_file` | File path to write logs to. | — |
| `log_source_location` | Include source file, line number, and method name in log entries. Adds a small runtime cost. | `false` |
| `trust_bundle_path` | Path to the SPIRE Server CA bundle file used for bootstrapping. | — |
| `trust_bundle_url` | URL to download the initial SPIRE Server trust bundle. Must use `https://` unless `trust_bundle_unix_socket` is also set. | — |
| `trust_bundle_unix_socket` | Unix socket to use when making the request specified by `trust_bundle_url`. When set, the URL must use `http://`. | — |
| `trust_bundle_format` | Format of the initial trust bundle: `pem` or `spiffe`. | `pem` |
| `join_token` | Optional join token generated by the SPIRE Server. | — |
| `join_token_file` | Path to a file containing an optional join token generated by the SPIRE Server. | — |
| `insecure_bootstrap` | Bootstrap without verifying the server's identity. Not recommended for production. | `false` |
| `rebootstrap_mode` | Rebootstrapping behavior: `never`, `auto`, or `always`. See [Rebootstrapping](#rebootstrapping). | `never` |
| `rebootstrap_delay` | Time to wait after detecting an untrusted server before beginning rebootstrapping. | `10m` |
| `workload_x509_svid_key_type` | Key type for workload X.509 SVIDs: `rsa-2048`, `ec-p256`, or `ec-p384`. | `ec-p256` |
| `availability_target` | Minimum desired availability during server or agent downtime. Influences how aggressively X.509 SVIDs rotate. Must be at least `24h` if set. See [Availability target](#availability-target). | — |
| `sds` | Optional SDS configuration block. See [SDS configuration](#sds-configuration). | — |
| `admin_socket_path` | Path to bind the admin API socket (disabled by default). | — |
| `authorized_delegates` | List of SPIFFE IDs allowed to use the Delegated Identity API. | — |
| `allow_unauthenticated_verifiers` | Allow the agent to release trust bundles to unauthenticated verifiers. | `false` |
| `allowed_foreign_jwt_claims` | List of trusted claims to return when validating foreign JWT-SVIDs. | — |
| `x509_svid_cache_max_size` | Soft limit on the number of X.509-SVIDs stored in the LRU cache. | `1000` |
| `jwt_svid_cache_max_size` | Hard limit on the number of JWT-SVIDs stored in the LRU cache. | `1000` |
| `profiling_enabled` | Enable the `net/http/pprof` endpoint. | `false` |
| `profiling_port` | Port for the pprof endpoint. Only used when `profiling_enabled` is `true`. | — |
| `profiling_freq` | Frequency for dumping profiling data to disk. Only active when `profiling_enabled` is `true` and this value is greater than `0`. | — |
| `profiling_names` | List of profile names to dump on each tick. See [Profiling names](#profiling-names). | — |
| `experimental` | Experimental options subject to change. See [Experimental options](#experimental-options). | — |

## Server attestation

The agent must establish a trusted connection to the SPIRE Server. This requires a bootstrap trust bundle in one of three forms:

<AccordionGroup>
  <Accordion title="trust_bundle_path (recommended for production)">
    The agent reads the bootstrap trust bundle from a file on disk. You must distribute this file to the node before starting the agent.

    ```hcl
    agent {
        trust_bundle_path = "/opt/spire/conf/initial_bundle.crt"
    }
    ```
  </Accordion>
  <Accordion title="trust_bundle_url">
    The agent downloads the bootstrap trust bundle from a URL. The URL must use `https://` and the server certificate must be valid against the system trust store. This simplifies deployment when distributing a file manually is impractical.

    ```hcl
    agent {
        trust_bundle_url = "https://spire-server.example.org/bundle.crt"
    }
    ```

    If `trust_bundle_unix_socket` is also set, the URL must use `http://` and the request is made against the specified local socket:

    ```hcl
    agent {
        trust_bundle_url         = "http://localhost/bundle.crt"
        trust_bundle_unix_socket = "/run/bundle-sidecar.sock"
    }
    ```
  </Accordion>
  <Accordion title="insecure_bootstrap (development only)">
    The agent connects to the server without verifying its identity. A man-in-the-middle attacker could control the SPIRE infrastructure in this mode.

    ```hcl
    agent {
        insecure_bootstrap = true
    }
    ```

    <Warning>
    `insecure_bootstrap` should only be used for testing and development. Never use it in production.
    </Warning>
  </Accordion>
</AccordionGroup>

Only one of `trust_bundle_path`, `trust_bundle_url`, or `insecure_bootstrap` may be set at a time.

## Rebootstrapping

Rebootstrapping re-establishes agent trust after it is lost — for example, after extended server downtime or a server reinstallation that could not preserve trust bundle continuity.

`rebootstrap_mode` accepts one of three values:

| Value | Behavior |
|---|---|
| `never` | Prevents automated rebootstrapping. Manual recovery is required if trust is lost. |
| `always` | Attempts rebootstrapping using the configured trust bundle source when needed. Fails the agent if the NodeAttestor plugin or server configuration is incompatible. |
| `auto` | Like `always`, but automatically disables rebootstrapping if the configuration is incompatible. |

`rebootstrap_delay` (default `10m`) sets the duration to wait after detecting an untrusted server before beginning rebootstrapping. If a trusted connection is established during this window, the clock resets.

- A longer delay reduces unnecessary rebootstrapping in environments where man-in-the-middle attacks are possible.
- A shorter delay allows faster recovery when the agent has been offline too long.

## SDS configuration

The agent supports [Envoy Secret Discovery Service (SDS)](https://www.envoyproxy.io/docs/envoy/latest/configuration/security/secret) over the same Unix socket as the Workload API.

```hcl
agent {
    sds {
        default_svid_name              = "default"
        default_bundle_name            = "ROOTCA"
        default_all_bundles_name       = "ALL"
        disable_spiffe_cert_validation = false
    }
}
```

| Option | Description | Default |
|---|---|---|
| `default_svid_name` | Resource name for the default X.509-SVID with Envoy SDS. | `default` |
| `default_bundle_name` | Resource name for the default X.509 bundle (trust domain CA certificates) with Envoy SDS. | `ROOTCA` |
| `default_all_bundles_name` | Resource name for all bundles, including federated trust domains, with Envoy SDS. Requires Envoy 1.18 or later for the SPIFFE Certificate Validator extension. | `ALL` |
| `disable_spiffe_cert_validation` | Disable the SPIFFE Certificate Validator extension in SDS validation contexts, falling back to standard X.509 chain validation. | `false` |

<Note>
Individual Envoy instances can override `disable_spiffe_cert_validation` by setting the key in their Envoy node metadata.
</Note>

## Availability target

<Note>
`availability_target` only affects agent SVIDs and workload X.509-SVIDs, not JWT-SVIDs.
</Note>

When `availability_target` is set, the agent rotates an X.509 SVID when its remaining lifetime drops to this duration. This makes rotation more aggressive to accommodate expected downtime.

To guarantee the target, the grace period — `SVID lifetime - availability_target` — must be at least 12 hours. If this condition is not met, the agent falls back to the default rotation strategy (rotating at half the SVID lifetime).

```hcl
agent {
    availability_target = "24h"
}
```

## Plugin configuration

Plugin configurations live in a top-level `plugins { ... }` block.

```hcl
plugins {
    pluginType "pluginName" {
        plugin_data {
            # plugin-specific options
        }
    }
}
```

| Option | Description |
|---|---|
| `plugin_cmd` | Path to the plugin binary. Not needed for built-in plugins. |
| `plugin_checksum` | Optional SHA-256 checksum of the plugin binary. Not needed for built-in plugins. |
| `enabled` | Enable or disable the plugin. |
| `plugin_data` | Inline plugin-specific configuration. Mutually exclusive with `plugin_data_file`. |
| `plugin_data_file` | Path to a file containing plugin-specific configuration. Mutually exclusive with `plugin_data`. |

<Note>
Plugins configured with `plugin_data_file` can be reconfigured at runtime (Posix only) by sending `SIGUSR1` to the SPIRE Agent process.
</Note>

### Plugin types

| Type | Description |
|---|---|
| `KeyManager` | Generates and stores the agent's private key. |
| `NodeAttestor` | Gathers information to attest the agent's identity to the server. |
| `WorkloadAttestor` | Introspects workloads to generate selectors for identity assignment. |
| `SVIDStore` | Stores X.509-SVIDs, bundles, and federated bundles into an external trust store. |

## Delegated Identity API

The Delegated Identity API allows an authorized workload to obtain SVIDs and bundles on behalf of workloads that cannot be attested by SPIRE Agent directly.

<Warning>
A workload authorized as a delegate gains the ability to impersonate any workload it can obtain SVIDs for. Grant delegate access only to highly trusted workloads.
</Warning>

To enable, set `admin_socket_path` and list the authorized SPIFFE IDs in `authorized_delegates`:

<Tabs>
  <Tab title="Unix">
    ```hcl
    agent {
        trust_domain = "example.org"
        admin_socket_path = "/tmp/spire-agent/private/admin.sock"
        authorized_delegates = [
            "spiffe://example.org/authorized_client1",
            "spiffe://example.org/authorized_client2",
        ]
    }
    ```
  </Tab>
  <Tab title="Windows">
    ```hcl
    agent {
        trust_domain = "example.org"
        experimental {
            admin_named_pipe_name = "\\spire-agent\\private\\admin"
        }
        authorized_delegates = [
            "spiffe://example.org/authorized_client1",
            "spiffe://example.org/authorized_client2",
        ]
    }
    ```
  </Tab>
</Tabs>

## Health check configuration

The agent can expose a health check endpoint by setting `listener_enabled = true`.

```hcl
health_checks {
    listener_enabled = true
    bind_address     = "localhost"
    bind_port        = "8080"
    live_path        = "/live"
    ready_path       = "/ready"
}
```

## Running as a Windows service

On Windows, SPIRE Agent can run as a Windows service. Only the `run` command is supported in service mode. SPIRE does not create the service automatically — you must register it manually.

```bash
sc.exe create spire-agent binpath=c:\spire\bin\spire-agent.exe
sc.exe start spire-agent run -config c:\spire\conf\agent\agent.conf
```
