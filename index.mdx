SPIRE (SPIFFE Runtime Environment) is a CNCF-graduated toolchain that automatically issues and rotates cryptographic identities to workloads across any infrastructure. It implements the [SPIFFE standard](https://spiffe.io) for establishing trust between services without secrets, passwords, or manual certificate management.

## How SPIRE works

<Steps>
  <Step title="Deploy SPIRE Server">
    The SPIRE Server is the root of trust for a trust domain. It manages registration entries, issues SVIDs, and maintains the trust bundle. Deploy a single server or a highly available cluster backed by a shared database.
  </Step>
  <Step title="Deploy SPIRE Agents">
    A SPIRE Agent runs on every node where workloads need an identity. The agent attests itself to the server using a node attestor plugin (AWS IID, GCP IIT, Kubernetes PSAT, join token, and more), then caches SVIDs for local workloads.
  </Step>
  <Step title="Register workload entries">
    Create registration entries on the server that map a SPIFFE ID to a set of workload selectors — Kubernetes pod labels, Unix UIDs, Docker image IDs, etc. The agent uses these to identify workloads at runtime.
  </Step>
  <Step title="Workloads call the Workload API">
    Workloads call the SPIFFE Workload API (served by the agent over a Unix socket) to receive their X.509-SVID or JWT-SVID. SVIDs are automatically rotated before expiry — no restarts required.
  </Step>
</Steps>

## Key capabilities

<CardGroup cols={3}>
  <Card title="Zero-trust identity" icon="shield-check">
    Every workload gets a cryptographic SPIFFE ID backed by a short-lived X.509 certificate or JWT — no shared secrets.
  </Card>
  <Card title="Multi-platform attestation" icon="cloud">
    Attest nodes on AWS, GCP, Azure, Kubernetes, bare metal (TPM, SSH, x509pop), and more via pluggable attestors.
  </Card>
  <Card title="Automatic rotation" icon="rotate">
    SVIDs and CA certificates are rotated continuously. Workloads receive updates via the streaming Workload API — no downtime.
  </Card>
  <Card title="Federation" icon="diagram-project">
    Federate multiple SPIRE deployments across organizations and cloud providers, enabling cross-domain mTLS without shared CAs.
  </Card>
  <Card title="Upstream CA integration" icon="certificate">
    Chain SPIRE into your existing PKI — HashiCorp Vault, AWS Private CA, GCP CAS, cert-manager, and more.
  </Card>
  <Card title="Envoy SDS" icon="network-wired">
    Integrate with Envoy's Secret Discovery Service to transparently deliver mTLS certificates to your service mesh.
  </Card>
</CardGroup>
